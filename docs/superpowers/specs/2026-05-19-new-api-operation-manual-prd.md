# NewAPI 操作手册 PRD

> 日期：2026-05-19  ｜  状态：待评审  ｜  版本：v1

## 1. 背景与目标

### 业务背景
NewAPI 是一个功能丰富的大模型网关系统，支持 40+ 种 AI 供应商接入，并具备细粒度的计费控制能力。然而，由于其配置项较多（如分组倍率、模型倍率、模型映射等），管理员在初次使用或进行复杂配置时容易出现操作失误。

### 北极星指标
- **文档覆盖率**：核心功能（渠道、计费）的操作说明覆盖率 100%。
- **用户满意度**：管理员配置流程的自愈能力（无需技术支持即可完成配置）显著提升。

## 2. 用户与场景

### 目标用户画像
- **系统管理员**：负责整体系统的初始化、供应商接入及费率调优。
- **运维人员**：负责日常渠道维护、余额监控。

### 核心场景
1. **供应商接入**：管理员需要将新购买的 Claude API 接入系统并分配给特定分组。
2. **费率调整**：因上游调价，管理员需要更新所有 gpt-4 系列模型的计费倍率。
3. **客户优惠**：为吸引大客户，管理员需要为 "enterprise" 分组设置特殊的计费权重。

## 3. 候选方案

### 方案 A：任务驱动型手册 (推荐)
- **描述**：按照“我要做什么”来组织内容（例如：如何添加一个 OpenAI 渠道）。
- **优点**：上手快，符合直觉。
- **缺点**：对于底层逻辑（如倍率叠加计算）的解释可能分散在各处。

### 方案 B：模块参考型手册
- **描述**：按照系统菜单模块组织内容（渠道管理、计费设置、用户管理）。
- **优点**：结构清晰，方便作为字典查询。
- **缺点**：跨模块的联动逻辑（如渠道与分组的关系）不易理解。

## 4. 推荐方案与理由

**推荐方案：混合结构方案**
- **理由**：兼顾快速上手与深度理解。先通过概念章节解释计费公式，再通过任务章节指导具体操作。
- **量化对比**：
  - 业务价值：高（解决核心操作痛点）
  - 技术成本：低（纯文档产出）
  - 用户影响：高（直接影响管理员效率）

## 5. 功能详述（手册大纲）

### 第一部分：渠道管理 (Channels)
- **基本概念**：什么是渠道？什么是优先级与权重？
- **操作指南**：
  - 创建渠道：详细说明类型、名称、密钥、代理等字段含义。
  - 批量操作：测试所有渠道、自动禁用处理。
  - 模型映射：如何将上游非标准模型名映射为标准名。

### 第二部分：计费与费率 (Pricing & Billing)
- **核心逻辑**：计费公式详解 `(PromptTokens * ModelRatio + CompletionTokens * ModelRatio * CompletionRatio) * GroupRatio`。
- **操作指南**：
  - 设置模型倍率 (Model Ratio)：针对不同模型的成本定价。
  - 设置分组倍率 (Group Ratio)：针对不同用户等级的定价体系。
  - 特殊设置：补全倍率 (Completion Ratio) 与缓存倍率 (Cache Ratio)。

### 第三部分：常见问题与排错
- 渠道测试失败的常见原因。
- 为什么用户余额扣除不符合预期？

## 6. 指标定义
- **准确性**：文档内容与 v1.0.0+ 版本代码逻辑完全一致。
- **易读性**：提供至少 3 个具体的 JSON 配置示例（如渠道额外设置）。

## 7. 合规与监管
- 提醒管理员在配置渠道时遵守上游服务商（如 OpenAI）的区域政策。
- 敏感信息（API Key）在手册示例中必须使用掩码处理。

## 8. 依赖与跨团队对齐
- 依赖：需要架构师确认 `relay` 模块中最新的倍率计算优先级逻辑。
- 对齐：已与项目管理专家对齐手册覆盖范围。

## 9. 风险与回退
- **风险**：代码更新导致文档过时。
- **对策**：手册应标注适用的版本范围，并建立文档随代码更新的机制。

## 10. 排期与版本
- **优先级**：P0
- **迭代**：第一版聚焦于费率与渠道，后续版本增加令牌管理与数据看板说明。
